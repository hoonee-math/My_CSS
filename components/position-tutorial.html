📍 Position 완벽 가이드

1️⃣ Static (기본값)

✏️ 아무 position도 안 주면 static
✏️ top, left, right, bottom 무시됨
✏️ 문서 흐름 그대로 배치
박스 1 (static)
박스 2 (static)
박스 3 (static)

2️⃣ Relative

✏️ 원래 자기 위치를 기준으로 이동
✏️ 원래 공간은 그대로 차지 (다른 요소에 영향 X)
✏️ top, left로 위치 조정 가능
박스 1 (static - 비교용)
박스 2 (relative: left 30px, top 10px)
박스 3 (static - 비교용)

👆 박스 2가 이동했지만, 박스 3은 원래 자리에 있음 (박스 2의 원래 공간이 유지됨)

3️⃣ Absolute (가장 중요!)

✏️ 가장 가까운 relative 부모를 기준으로 위치 잡음
✏️ 부모에 relative가 없으면? → 화면(body) 기준으로 배치됨
✏️ 문서 흐름에서 빠져나옴 (공간 차지 안 함)

✅ 올바른 사용법

부모 (position: relative) - 점선 박스
자식 (position: absolute)
top: 10px, right: 10px

✅ 부모가 relative → 자식 absolute가 부모 안에서 위치 잡음

❌ 부모가 relative 없으면?

부모 (position 없음 = static)
자식 (absolute) - 화면 기준으로 배치됨!

❌ 부모가 static → 자식이 화면(body) 기준으로 튀어나감

6️⃣ Team Card 패턴 (3개 레이어)

💡 당신의 team-card.html과 같은 구조!

Layer 3: 텍스트

position: relative + z-index: 2

가장 위에 표시됨 
/* Layer 1: 배경 */
.layered-box {
    position: relative;  /* ← ::before의 기준점 */
}

/* Layer 2: 어두운 오버레이 */
.layered-box::before {
    position: absolute;  /* ← layered-box 기준 */
    top: 0; left: 0; right: 0; bottom: 0;
    background: rgba(0,0,0,0.3);
    z-index: 1;
}

/* Layer 3: 텍스트 */
.layered-content {
    position: relative;  /* ← z-index 적용하려면 필요 */
    z-index: 2;  /* ← ::before 위에 표시 */
}

📌 핵심 정리

언제 사용할까?

Position 언제 사용? 특징
static 기본값 (명시 안 해도 됨) 일반적인 흐름
relative 1. absolute의 기준점 만들기
2. 살짝 위치 조정 		원래 공간 유지
absolute 1. 뱃지, 아이콘 배치
2. 오버레이
3. 툴팁 		부모(relative) 기준 배치

황금 공식

부모: relative + 자식: absolute = 완벽한 조합 ✨

team-card.html에서

  • team-card relative → ::before의 기준점
  • team-card::before absolute → team-card를 꽉 채움
  • team-card-content relative → z-index로 위에 표시
